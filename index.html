Unit 1: Basics of Computer

Introduction · Characteristics · History · Generations · Classification

What is a Computer?

Computer — an electronic machine that accepts data, processes it, and generates output. The term comes from "compute" (to calculate).

Key Idea: A computer performs both simple and complex operations with speed and accuracy.

Scenario: Online Exam System

Ten thousand students take an exam online. The computer must quickly calculate scores, generate individual reports, and store each student's data without errors. This demonstrates the computer's speed, accuracy, and storage capability.

Scenario: Bank ATM Network

A bank customer withdraws money in London. The ATM communicates with the main database to verify the balance, deduct the amount, and update the ledger instantly. This shows the computer's versatility and diligence in a multi-user environment.

Unit 2: Computer System Hardware

CPU · ALU · Registers · Motherboard · Buses · Expansion Slots

Central Processing Unit (CPU)

CPU — the "brain" of the computer. It consists of the Arithmetic Logic Unit (ALU) and the Control Unit (CU), plus a set of registers for temporary storage.

CPU: ALU + CU + Registers (Heart of the computer)

ALU (Arithmetic Logic Unit)

Performs arithmetic (+, -, ×, ÷) and logic operations (comparisons: <, >, =).

Registers

High-speed storage areas inside the CPU. They hold data, instructions, and intermediate results during processing.

Scenario: Video Editing

When you edit a 4K video, the CPU's ALU processes millions of pixel calculations per second. The registers temporarily store the color values for each pixel, while the Control Unit directs the flow of data from the hard drive to the GPU and RAM.

Motherboard & Buses

Motherboard — the main circuit board that connects all components. It contains the CPU socket, RAM slots, expansion slots (PCI, AGP, PCIe), and I/O ports.

System Bus — set of pathways for data, addresses, and control signals. It includes:

  • Data Bus — transfers data between CPU and memory.
  • Address Bus — carries memory addresses.
  • Control Bus — carries read/write commands and status signals.
Motherboard with CPU, RAM, expansion slots (The hub of the computer)

Unit 3: Computer Memory

Memory Hierarchy · RAM · ROM · Cache · Secondary Storage

Memory Representation

Bit — smallest unit (0 or 1).
Byte — 8 bits, the smallest addressable unit.
Kilobyte (KB) = 1024 bytes · Megabyte (MB) = 1024 KB · Gigabyte (GB) = 1024 MB · Terabyte (TB) = 1024 GB.

Scenario: Opening a Large Game

When you launch a modern game (100 GB), the operating system loads 4 GB of frequently used data into RAM for fast access. The cache stores the characters, textures, and commands you use most often during gameplay, ensuring smooth performance.

RAM vs ROM

RAM (Random Access Memory)

  • Volatile (losses data on power off)
  • Read/write
  • DRAM (Dynamic) — main memory
  • SRAM (Static) — cache memory

ROM (Read Only Memory)

  • Non-volatile (retains data)
  • Read only (typically)
  • Stores BIOS / firmware
  • PROM, EPROM, EEPROM variants

Unit 4: Input & Output Devices

Keyboards · Pointing Devices · Scanners · Printers · Monitors · Ports

Input Devices

Keyboard

Standard QWERTY layout. Keys: typing, numeric, function, control, special.

Mouse

Pointing device. Physical (ball) or optical. Actions: point, click, right-click, double-click, drag & drop.

Joystick

Used for gaming and controlling cursor speed.

Digitizing Tablet

For CAD, drawing, and sketching. Pen + tablet.

Light Pen & Touch Screen

Direct interaction with screen. Used in ATMs, kiosks.

Scanner

Converts paper documents to digital images. Hand-held or flat-bed.

Scenario: Airport Check-in

At the airport, a clerk uses a keyboard to type your passport number and a mouse to select the correct flight. A barcode scanner reads your boarding pass, and a thermal printer prints a luggage tag. The monitor displays the seat assignment and gate information.

Output Devices

Monitor

CRT or LCD. Resolution (pixels), dot pitch, refresh rate.

Printer

Impact (Dot matrix, Daisy wheel, Drum) — low cost, bulk printing.
Non-impact (Ink-jet, Laser) — high quality, quiet.

Plotter

Vector graphics output for maps, blueprints, CAD/CAM.

Audio Output

Speakers, headphones. Sound card converts digital to analog.

Video Output

Data projector for presentations.

Unit 5: User-Computer Interface (Software)

System Software · Operating System · Device Drivers · Utilities · Programming Languages

Types of Software

System Software

Provides basic functionality, controls hardware, acts as interface. Examples: Operating System (Windows, Linux), device drivers, utilities.

Application Software

Performs specific tasks for users. Examples: MS Word, Excel, Photoshop, Chrome, Tally.

Scenario: Using a Smartphone

When you open your smartphone, the operating system (android/iOS) starts up first, managing the camera, fingerprint sensor, and Wi-Fi. Then you open the application software (WhatsApp, YouTube, Camera) to send a message or take a photo. The device drivers translate your touches into commands the CPU can understand.

Unit 7: Computer Networks

Data Transmission · LAN · MAN · WAN · Topologies · Network Devices

Data Transmission Media

Guided (Wired)

  • Twisted Pair (UTP/STP) — common, cheap, up to 100m
  • Coaxial — thicker, longer distance (185–500m)
  • Optical Fiber — light signals, high speed, long distance, secure

Unguided (Wireless)

  • Radio (RF) — broadcast, no physical connection
  • Microwave — directional, line-of-sight (≈42km)
  • Satellite — geosynchronous, global coverage

Scenario: Watching Netflix

You stream a movie at home. The optical fiber from your ISP brings the video data to the neighborhood, then the coaxial cable carries it to your house. Your Wi-Fi router (radio waves) transmits the data to your tablet. This is a combination of guided and unguided media.

LAN Topologies

Bus

All devices on a central cable (bus). Easy to implement, but cable failure brings down the network.

Ring

Devices connected in a circle. Token passing for access. Failure of one node affects the ring.

Star

All devices connect to a central hub/switch. Most popular. Hub failure disables the network.

Unit 8: Internet Basics

History · WWW · Browser · URL · ISP · E-mail · Search Engines · E-commerce

Evolution of Internet

1969 — ARPANET (US Department of Defense) — first network.
1980s — NSFNET, Usenet, e-mail.
1990s — World Wide Web (WWW) developed at CERN by Tim Berners-Lee. Explosive growth.

Scenario: Online Shopping

You open an e-commerce website. The URL (amazon.com) is resolved by a DNS server to an IP address. Your browser sends an HTTP request, and the web server responds with the page HTML. The search engine helps you find the product, and you complete the purchase using encryption for security.

Key Internet Terms

World Wide Web (WWW)

Collection of linked documents (web pages) accessed via the Internet using HTTP.

Web Page / Website

A web page is a single HTML document. A website is a set of related pages.

URL (Uniform Resource Locator)

Example: http://www.xyz.com/tutor/start/main.htm
Protocol → Domain Name → Path → File Name

Domain Types

.com (commercial), .edu (education), .gov (government), .org (non-profit)

Unit 10: Computer Security

Security Attacks · Malicious Software · Cryptography · Digital Signatures · Firewalls · Authentication

Security Threats

Malicious Software

  • Virus — attaches to programs, replicates, destroys data.
  • Worm — self-replicating, uses network holes, consumes resources.
  • Trojan Horse — masquerades as useful software, hides destruction.
  • JavaScript / ActiveX — malicious scripts embedded in web pages.

Hacking

  • Packet Sniffing — intercepts data packets over the network.
  • Password Cracking — attempts to break encrypted passwords.
  • E-mail Hacking — reading or altering e-mail in transit.
  • Denial of Service (DoS) — floods a system with requests, making it unavailable.

Scenario: Phishing Email

You receive an email pretending to be from your bank, asking for your password. This is a Trojan horse attack. Knowing this, you use encryption and two-factor authentication to protect your account. A firewall would also block suspicious traffic.

Authentication Methods

  • Username & Password — most common. Weak if simple.
  • Smart Card — embedded chip, stores credentials.
  • Biometrics — fingerprint, iris/retina scan, voice pattern, facial recognition.

Information Systems

Data · Information · Knowledge · CBIS · TPS · MIS · DSS · EIS · ERP · E-commerce

Data, Information & Knowledge

Data

Raw facts, unprocessed. Example: "24th", "Oct", "2012"

Information

Processed, organized data with meaning. Example: "Independence Day on 24th Oct 2012"

Knowledge

Inference or conclusion drawn from information. Example: "Plan trip to Kitwe for the celebrations."

Scenario: Inventory Management

A store uses a TPS to track each sale (data). At the end of the day, the MIS generates a report showing total sales and low-stock items (information). The manager uses this knowledge to reorder supplies and adjust prices, ensuring the store runs efficiently.

Types of Information Systems
